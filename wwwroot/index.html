Processing Bookmarks

If the document contains a named anchor tage (i.e. a bookmark) like

<a name="somename">...

and the bookmark contains content then nothing will happen. The browser will be left to do what browsers do, in this case load the page and scroll the content marked into view (if necessary!)

If the bookmark exists, but is empty, then the navControl library will attempt to find content. It does this by looking at the name of the bookmark (everything after the # on the url) and then atempts to load content from a sub-folder called "./content"

The file name to be searched for is [[name]].html

If the file is found an attempt to load its contents using the jQuery load method (see jQuery documentation)

                    var sBookmarkSelector = "a[name='" + sBookmark + "']";
                    var urlContent = "content/" + sBookmark + ".html";
                    var tagBookmark = $(sBookmarkSelector).get(0);
                    ...
                    $.ajax(urlContent, {
                    success: function (data, textStatus, jqXHR) {
                    //assume that the data is valid HTML
                    tagBookmark.innerHTML = data;
                    },
                    error: function (jqXHR, textStatus, errorThrown) {
                    console.log("Failure to load requested content from " + urlContent);
                    }
                    })
